# DevDigest

Local-first AI pull-request review studio. Course **starter**: does one thing
end to end — import a PR and run an agent review on it. Later lessons add features.

## Layout — where things live

| Folder           | Package                    | What it is                                   | Port |
|------------------|----------------------------|----------------------------------------------|------|
| `server/`        | `@devdigest/api`           | Fastify API + Drizzle/Postgres (pgvector)    | 3001 |
| `client/`        | `@devdigest/web`           | Next.js 15 web app (the studio)              | 3000 |
| `reviewer-core/` | `@devdigest/reviewer-core` | Pure engine: diff → prompt → LLM → findings  | —    |
| `e2e/`           | `@devdigest/e2e`           | Deterministic browser e2e (agent-browser)    | —    |

Shared Zod contracts: `server/src/vendor/shared` (`@devdigest/shared`).
`repo-intel` (codebase indexer feeding review context) lives in
`server/src/modules/repo-intel`.

## Stack

Node ≥22 · **pnpm ≥10** (server, client) · **npm** (reviewer-core, e2e) · Docker (Postgres only).
Key versions: Fastify 5 · Drizzle 0.38 · Next 15.1 · React 19 · TanStack Query 5 · Tailwind 4.

## Commands

- `./scripts/dev.sh` — boot everything from zero (docker → migrate → seed → API + web).
  Flags: `--no-seed` · `--no-client` · `--db-only`.
- Per package: `dev` · `test` · `typecheck` (+ `build`, `db:migrate`, `db:seed` in server).
- Server tests split: unit `pnpm exec vitest run --exclude '**/*.it.test.ts'`;
  integration (needs Docker) `pnpm exec vitest run .it.test`.

## Non-default conventions

- **Not a workspace.** Each package has its own `package.json`/lockfile; cross-package
  code is shared via tsconfig path aliases, not published modules.
- **`reviewer-core` is imported as raw TypeScript source** (alias `@devdigest/reviewer-core`),
  not built JS. Its `build` is `tsc --noEmit` (emits nothing).
- **Package managers differ**: server + client use pnpm; reviewer-core + e2e use npm.
- **Migrations are NOT run on boot** — apply them manually.
- `server/package.json` is `skip-worktree` (a local variant diverges); CI invokes the
  test split via `pnpm exec vitest run …`, not committed `test:unit`/`test:integration`.

## Gotchas

- `relation … does not exist` on first run → migrations weren't applied:
  `cd server && pnpm db:migrate`.
- API crashes at boot with `ERR_MODULE_NOT_FOUND` → `reviewer-core/node_modules`
  is missing (`cd reviewer-core && npm ci`).
- Port 5432 in use → another Postgres; stop it or change the host port in `docker-compose.yml`.

## Do not touch

- `server/clones/**` — runtime git clones (gitignored). Never edit or commit.
- `server/src/db/migrations/**` — generated by drizzle-kit; change the schema, then regenerate.

## Read when

- New to the repo / architecture → read `README.md`.
- Working inside a package → read that package's `AGENTS.md` (e.g. `server/AGENTS.md`).
- Writing or editing a reviewer agent's prompt → read `docs/agent-prompts/`.
- Testing / CI strategy → read `TESTING.md`.
