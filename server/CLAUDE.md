# @devdigest/api (server)

Fastify API + Drizzle/Postgres (pgvector). Runs on the host (`pnpm dev`), :3001.

## Stack

Fastify 5 · Drizzle ORM 0.38 · Postgres + pgvector · Zod · tsx (no build step).
Package manager: **pnpm**.

## Commands

- `pnpm dev` — tsx watch, :3001.
- `pnpm exec vitest run --exclude '**/*.it.test.ts'` — unit (hermetic, no Docker).
- `pnpm exec vitest run .it.test` — integration (real Postgres via testcontainers, needs Docker).
- `pnpm db:migrate` · `pnpm db:seed` · `pnpm db:generate` · `pnpm typecheck`.

## Conventions

- **A module = a self-contained Fastify plugin** under `src/modules/<name>`
  (routes + service + repository), registered statically in `src/modules/index.ts`.
- **Adapters come from the DI container** (`src/platform/container.ts`) — call
  `container.llm()`, `.github()`, `.git()`, etc. Never import an adapter directly.
- **Zod drives both request validation and response serialization** (one schema, both ends).
- **Secrets only via `SecretsProvider`** → `~/.devdigest/secrets.json` (mode 0600).
  Never put API keys/tokens in the DB, git, or `AppConfig`.
- Business logic lives in services; routes are thin; data access goes through repositories.

## Gotchas

- Migrations are **manual** — not run on boot. `relation … does not exist` ⇒ run `db:migrate`.
- `*.it.test.ts` files need Docker (testcontainers); the unit lane excludes that glob.
- `package.json` is `skip-worktree` — a local variant diverges from the committed file.

## Do not touch

- `clones/**` — runtime git clones (gitignored).
- `src/db/migrations/**` — generated by drizzle-kit; edit the schema, then regenerate.
